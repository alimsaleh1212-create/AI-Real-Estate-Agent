# AI Real Estate Agent

AI-powered home price estimator for Ames, Iowa. Describe a property in plain English — the app extracts features with Gemini LLM and predicts the sale price with a trained GradientBoosting model.

---

## Features

- **Natural language input** — describe a property; the LLM extracts the 10 key features
- **Gap filling** — a review form lets you correct or complete any missed features
- **Price prediction** — GradientBoosting pipeline (CV RMSE 0.1468, Test R² 0.9149)
- **Plain-English interpretation** — Stage 2 LLM explains the prediction in context
- **Market insights** — ask analysis questions ("What is the median price in Ames?")
- **Analytics dashboard** — interactive charts: distributions, correlations, neighbourhood medians, feature importance
- **Predictions log** — history of past predictions (requires Supabase)
- **REST API** — FastAPI backend with `/predict`, `/insights`, `/health` endpoints

## Quick Start

### Prerequisites

- Python 3.11+
- [`uv`](https://docs.astral.sh/uv/) — `curl -LsSf https://astral.sh/uv/install.sh | sh`
- A [Google Gemini API key](https://aistudio.google.com/) (free tier)

### 1. Clone and install

```bash
git clone <repo-url>
cd project2_ai_real_estate_agent
uv sync
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env and set GOOGLE_API_KEY
```

### 3. Run the Streamlit UI

```bash
uv run streamlit run ui/app.py
```

Open [http://localhost:8501](http://localhost:8501).

### 4. Run the FastAPI backend (optional)

```bash
uv run uvicorn app.main:app --reload
```

API docs at [http://localhost:8000/docs](http://localhost:8000/docs).

## Docker

Build and start both services with a single command:

```bash
# Copy and fill in your API key first
cp .env.example .env

docker compose up --build
```

| Service | URL |
|---|---|
| Streamlit UI | http://localhost:8501 |
| FastAPI | http://localhost:8000 |
| API docs | http://localhost:8000/docs |

To stop:

```bash
docker compose down
```

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `GOOGLE_API_KEY` | **Yes** | Gemini API key from [aistudio.google.com](https://aistudio.google.com/) |
| `SUPABASE_URL` | No | Supabase project URL for prediction logging |
| `SUPABASE_KEY` | No | Supabase anon key |

The app runs fully without Supabase — prediction/insight logging is silently skipped when those variables are absent.

## Model

The production model is a scikit-learn `Pipeline` with a `ColumnTransformer` preprocessor and a `GradientBoostingRegressor`.

**10 selected features** (chosen by Pearson, Spearman, ANOVA F-test, and Mutual Information):

| Feature | Type | Description |
|---|---|---|
| `OverallQual` | Numeric | Overall material and finish quality (1–10) |
| `TotalSF` | Numeric | Total floor area: basement + 1st + 2nd floor (sqft) |
| `GarageCars` | Numeric | Garage capacity (number of cars) |
| `TotalBath` | Numeric | Total bathrooms (full + 0.5 × half) |
| `YearBuilt` | Numeric | Original construction year |
| `TotalBsmtSF` | Numeric | Total basement area (sqft) |
| `KitchenQual` | Ordinal | Kitchen quality (Po / Fa / TA / Gd / Ex) |
| `BsmtQual` | Ordinal | Basement quality (None / Po / Fa / TA / Gd / Ex) |
| `ExterQual` | Ordinal | Exterior material quality (Po / Fa / TA / Gd / Ex) |
| `Neighborhood` | Nominal | Physical location within Ames city limits |

**Metrics on held-out test set:**

| Metric | Value |
|---|---|
| CV RMSE (log scale) | 0.1468 |
| Test R² | 0.9149 |
| Test MAE | ~$14,900 |

To retrain:

```bash
uv run python -m src.ml_pipeline
```
